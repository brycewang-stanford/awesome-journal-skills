# 政策含义（cpa-policy-implication）

## 触发时机

- 文末是"强化 / 完善 / 健全 / 推进"四件套
- 政策建议很多，但每一条都和本文证据无关、放到任何论文都成立
- 审稿人指出"对策建议空泛，未建立在分析之上"

## 《中国行政管理》的定位：可操作，但必须有据

与经济学期刊（《经济研究》偏"意义层 / 制度层启示"）不同，《中国行政管理》面向政府实践，政策建议**可以也应该更具体、更可操作**。但这把双刃剑的另一面是：**每一条建议都必须从本文的发现 / 机制直接推出**，否则就退化成"对策套话"。

判断红线：**把本文的实证 / 案例发现遮住，这条建议还成立 → 它就是空话，删掉或重写。**

## 从发现到建议的推导链

每条政策含义都走"发现 → 机制 → 建议"三步：

```
本文发现：[具体发现，带方向 / 条件]
→ 这是因为：[本文识别的机制 / 过程]
→ 因此建议：[针对该机制的、指明"谁在哪个环节做什么"的具体措施]
```

反例（断链）：发现"数字化考核加重基层负担"，却建议"加强基层党的建设"——发现与建议无逻辑关联。

## 结构（推荐三段，每段一条推导链）

```
本文的研究具有以下政策含义。

第一，[针对核心发现]——本文发现……，这一效应主要通过[机制]实现。
因此，[某层级 / 某部门]应在[具体环节]……，重点关注[本文识别出的高风险情境 / 群体]。

第二，[针对异质性 / 条件]——本文进一步发现效应在[某情境]下更强。
这意味着政策应[差异化 / 分类施策]，对[该情境]……。

第三，[针对边界 / 风险]——本文也发现……，提示在推进[某项改革]时应防范……。
```

## 可操作 ≠ 假大空

| 空泛（删 / 改） | 可操作且有据（替换为） |
|---------------|---------------------|
| "加强基层治理" | "针对本文识别的'考核挤占入户时间'机制，建议上级在数字化考核中设置留痕豁免清单，将一线服务时长纳入考核" |
| "完善央地关系" | "针对层层加码，建议在目标责任书中明确属地不得自行加码的事项边界" |
| "推进数字政府建设" | "针对本文发现的应用泛滥，建议建立政务 App 准入与定期清理机制" |

具体到**主体（谁）+ 环节（在哪）+ 动作（做什么）**，且能追溯到本文哪个发现。

## 必查清单

- [ ] 每条政策含义都能追溯到本文具体发现 / 机制（不是放之四海皆准）
- [ ] 不出现"强化 / 完善 / 健全 / 推进 / 加强 / 重视"撑满段落
- [ ] 建议指明"谁 + 在哪个环节 + 做什么"
- [ ] 与异质性发现衔接，体现分类 / 差异化施策
- [ ] 提及本文发现的边界 / 风险，不只报喜
- [ ] 不与本文证据脱节（不夸大、不外推到未研究的领域）

## 反模式

- "对策建议"四件套（强化 / 完善 / 健全 / 推进）撑满整节，无一条挂得上证据
- 建议比发现还多、还细，像政策咨询报告而非学术论文的含义部分
- 把政策建议写成对上级文件的"学习体会"或表态
- 文末才提到中国治理——应贯穿全文的问题意识
- "本文是首次……"——这是贡献，不是政策含义

## 输出格式

```
【建议条数】X
【推导链完整的条数】X / 总数（每条能否追溯到发现）
【四件套命中】X 处
【可操作性】具体到主体+环节+动作 / 偏空泛
【与异质性衔接】是 / 否
【下一步】cpa-abstract
```
