# 语言风格（cpa-style）

## 触发时机

- 通读初稿后觉得"读起来像政策文件 / 工作汇报 / 学习体会"
- 一段里出现 ≥ 2 个"具有重要意义" / "为……提供新视角"
- 政策建议读起来像"强化、完善、健全、推进"四件套
- 理论框架"开头点一下、后文再不出现"，分析与理论两张皮

## 对照表（黑名单 → 白名单）

| 问题类型 | 黑名单（删 / 改） | 白名单（替换为） |
|----------|------------------|----------------|
| 空洞意义 | "具有重要的理论价值和现实意义" | "本文为理解[现象]提供了[理论框架]下的解释，并识别出[机制]" |
| 四件套对策 | "应强化 / 完善 / 健全 / 推进……" | "[某层级]应在[环节]建立[机制]，重点针对本文识别的[情境]" |
| 理论两张皮 | 开头提理论、后文与分析无关 | 分析每一步回扣理论：用[理论概念]解释观察到的[现象] |
| 案例只叙事 | "调研发现该地做了 A、B、C" | "该地的 A、B 行为印证了[机制]——因为[理论预期]" |
| 表态式 | "本文深入贯彻……精神" | 删除——学术论文不是表态材料 |
| 贡献模糊 | "丰富了相关研究" | "区别于[已有研究]的 X 视角，本文首次从 Y 揭示了 Z" |
| 数字无解读 | "表 3 报告了回归结果" | "表 3 显示，数字化考核强度每提高一个标准差，负担上升 0.09 个标准差" |
| 概念堆砌 | 连续辨析多个相近概念不落地 | 选定本文用的概念定义，说明为何采此定义、与分析如何衔接 |

## 行文规范

### 理论—分析一致性（公共管理稿件的关键）

每一处经验 / 案例分析都应**回扣到理论框架的概念**。审稿人快速读，若分析部分完全脱离开头的理论框架，会判为"理论是装饰"。做法：分析段落主动使用理论概念（如"自由裁量""注意力挤占""属地加码"）来命名观察到的现象。

### 人称与谦逊

- 避免"笔者""我们认为"出现在结论性段落；用"本文发现 / 研究表明 / 资料显示"
- 经验 / 案例研究只能说"本文提供了证据支持 / 本文的资料表明"，不要写"本文证明了"
- 不把统计显著当治理上的重要——同时说明**实践 / 治理意义上的量级**

### 一段一概括

每个段落第一句必须是**概括句**，后续句子是支撑。审稿人读完每段首句应能复原文章逻辑。

错误：
> 张三（2020）发现……。李四（2021）发现……。王五（2022）发现……。

正确：
> 关于执行偏差的成因，文献分为结构论与行动者论两派（张三，2020；李四，2021）。前者强调……，后者强调……。本文整合两者……。

## 行文自检清单

- [ ] 每段首句是概括性陈述
- [ ] 黑名单短语命中数 = 0
- [ ] 政策建议指明"谁在哪个环节做什么"，且挂得上发现
- [ ] 分析部分主动使用理论框架的概念（理论不是装饰）
- [ ] 案例 / 数据后有解读与回扣，不只陈述
- [ ] 无表态式、学习体会式语句
- [ ] 人称、谦逊度全文统一

## 反模式

- 通篇"具有重要的 X 意义"——一次都不应出现
- 政策建议用"强化、完善、健全、推进"四个动词撑满段落
- 开头理论框架华丽，正文分析与之无关
- 案例部分纯叙事，读不出理论命题
- 把论文写成对政策文件的解读 / 表态

## 输出格式

```
【黑名单命中】X 处，分别在：[...]
【段首概括率】X / 总段数
【理论-分析一致性】紧扣 / 两张皮 [...]
【四件套命中】X 处
【案例 / 数据解读】到位 / 仅陈述
【人称一致性】一致 / 待统一
【下一步】cpa-submission
```
