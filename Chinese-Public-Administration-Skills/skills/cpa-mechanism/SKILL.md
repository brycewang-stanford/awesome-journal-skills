# 治理 / 政策过程机制（cpa-mechanism）

## 触发时机

- 已有主结果（效应或现象），但说不清"如何发生"
- 机制只有"政策大概起了作用"式的口头说明
- 审稿人问"这个治理效果是通过什么过程实现的？"

## 公共管理常见机制谱系

机制要挂在公共管理理论上，不能凭空想。常见谱系：

1. **政策执行机制**——目标模糊—冲突、自由裁量、街头官僚、执行变通 / 选择性执行
2. **注意力分配机制**——领导注意力、议程设置、运动式治理、"中心工作"挤占
3. **央地关系机制**——属地管理、层层加码、向上负责 vs. 向下回应、政策再制定
4. **激励与问责机制**——晋升锦标赛、考核指标、问责压力、避责行为
5. **资源与能力机制**——财政能力、人员编制、数字 / 技术赋能
6. **协同与网络机制**——部门协同、政社关系、多元主体协作

## 两类机制证据

### 定量路径：中介 / 调节

- 不要只跑 Sobel 检验 → 推荐 Bootstrap + 因果中介框架；报告直接 / 间接效应与占比
- 或替换为机制变量：处理 → 机制变量 M（如考核压力、注意力指标）→ 结果 Y
- 机制变量构造的**时间逻辑**正确（在处理之后、结果之前），不使用未来信息

### 定性路径：过程证据

- 用过程追踪还原"处理 → 机制 → 结果"的因果链，每一环有资料佐证
- 机制的**观察含义**设检验：若机制成立应观察到什么？用箍式 / 冒烟枪检验评估证据强度
- 排除竞争性机制（同一结果的其他可能过程）

## 机制写作三段式

```
本文进一步从[机制名，挂某理论]角度揭示效应 / 现象的形成过程。
首先，从理论上看，[机制]应通过[路径]导致[结果]……
其次，[定量：本文用机制变量 M 检验，结果见表 X；或 定性：本文通过 A、B、C 三段过程证据还原该机制]……
最后，结合[已有文献 / 理论]，本文认为该机制在[中国治理情境]下表现为……。
```

## 机制设计三原则

1. **理论指引**：机制必须能挂上公共管理理论（执行 / 注意力 / 央地 / 激励问责）
2. **过程可见**：定量要有可测的中介变量，定性要有可追溯的过程证据，不能只是"我们推测"
3. **可证伪**：能写出"若机制成立 → 应观察到 X；若不成立 → 应观察到 Y"

## 必查清单

- [ ] 至少 1 个机制（理想 2 个互补机制，如结构机制 + 行动者机制）
- [ ] 机制来源于公共管理理论 / 已有文献（不是凭空想）
- [ ] 定量：机制变量时间逻辑正确；定性：每一环有资料编号佐证
- [ ] 排除了主要的竞争性机制
- [ ] 机制结论与主结果方向一致（或合理解释不一致）

## 反模式

- 用"政策起了作用 / 我们认为可能因为……"一句话代替机制分析
- 把"调节变量"当"机制"（调节回答在何条件下更强，不直接回答如何发生）
- 中介效应只跑 Sobel 检验
- 定性机制只有叙事，没有证据强度判断、没有排除竞争解释
- 机制变量构造使用了未来信息

## 输出格式

```
【机制数】X 个（结构 / 行动者）
【机制理论】执行 / 注意力 / 央地 / 激励问责 / 资源能力 / 协同网络
【证据类型】定量中介 / 定性过程 / 两者
【时间逻辑 / 证据链】通过 / 待修
【竞争性机制】已排除 / 待处理
【下一步】cpa-heterogeneity
```
