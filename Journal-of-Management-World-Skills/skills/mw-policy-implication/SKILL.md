# 政策启示（mw-policy-implication）

## 触发时机

- 文末政策建议只有一句"建议政府重视"
- 审稿人指出"政策含义太空"
- 准备投稿但结论段没写够

## 为什么是硬约束？

《管理世界》编委的政策导向极强。即使是理论文章，编委也会期待"对治理 / 政策 / 实践有什么启示"。**没有政策启示的实证文章命中率极低**。

## 分层结构

政策建议分两个维度分层：

### 维度 1：时间层

- **短期**（1–2 年）：政策微调、执行细则
- **中期**（3–5 年）：制度配套、跨部门协同
- **长期**（5 年以上）：基础制度建设

### 维度 2：层级层

- **微观**（企业 / 个体）：经营决策、合规建议
- **中观**（行业 / 地方）：行业自律、地方试点
- **宏观**（中央 / 部委）：法律法规、宏观调控

至少写 3 条政策建议，**分别覆盖不同的时间 × 层级组合**。

## 模板

```
第一，[短期 × 宏观]——基于本文发现，建议[具体部门]在[时间窗]内，[具体可执行动作]。
具体而言，可通过[路径 1] / [路径 2] 实现，预计效果为[本文实证结果]。

第二，[中期 × 中观]——……

第三，[长期 × 微观]——……
```

每条建议都必须：
- 指向**具体部门**（"政府"不够 → 财政部 / 发改委 / 证监会 / 地方人民政府）
- 给出**可执行动作**（不是"加强 / 重视"）
- 与本文**实证证据挂钩**（不能凭空建议）

## 必查清单

- [ ] 至少 3 条建议
- [ ] 覆盖至少 2 个时间层 + 2 个层级层
- [ ] 每条建议指向具体部门
- [ ] 每条建议给出可执行动作
- [ ] 每条建议挂钩本文证据

## 反模式

- "建议政府加强监管 / 重视问题 / 完善制度"——无效建议
- 建议与本文实证无关（写了 ESG 投入但建议讨论员工持股）
- 建议过于宏大（"建议改革资本市场"）
- 仅一段话堆叠所有建议，无层次

## 输出格式

```
【建议数】X 条
【时间层覆盖】短 / 中 / 长
【层级层覆盖】微 / 中 / 宏
【可执行性】高 / 中 / 低
【与本文证据挂钩度】高 / 中 / 低
【下一步】mw-submission
```
