# Physical Review Letters — Manuscript Skeleton

> A Letter is SHORT and makes ONE central claim. Everything below is budgeted against
> the deductible length limit (text + figures + equations + references). Verify the
> current limit and length formula on the official APS / PRL author page. The canonical
> source format is REVTeX; this Markdown skeleton mirrors that structure.

## Title

> Encode the central RESULT, not just the topic.
> [Concise result-bearing title]

## Authors and affiliations

> Author A,1 Author B,2,* ...
> 1 Institution, City, Country
> 2 Institution, City, Country

## Abstract

> Self-contained mini-Letter, intelligible to a physicist outside the subfield.
> Lead with the FINDING. Keep within the current abstract length limit.
> Pattern: context (1 sentence) → what we did (1 clause) → what we found (key result + number)
>          → why it matters broadly (1 sentence).

[Abstract text]

## Body

> Target ~3750 words / ~4 journal pages INCLUDING figures and references (verify).
> No section headings are typically used in a Letter; the flow below is logical, not literal.

### Opening (the gate paragraph)

> 4 moves, within ~1–2 paragraphs:
> 1. General physics context (non-specialist accessible)
> 2. The open question / gap
> 3. What this Letter shows — the central claim, with the key number/surprise
> 4. Why it matters broadly
[...]

### Setup / model (trust-minimum)

> One compact paragraph: the essential experimental configuration or theoretical setup.
> Full apparatus / full Hamiltonian → Supplemental Material.
[...]

### Defining equations (inline, ≤ a few)

> Only the equation(s) that DEFINE the claimed quantity. Interpret each physically.
> Derivation chains → SM. Remember: displayed equations cost word-equivalents.
[ Eq. (1) ]
[ Eq. (2) ]

### Central result + decisive control

> State the headline result. State the ONE control that rules out the obvious artifact.
> Quote the headline uncertainty (statistical + dominant systematic).
[...]
> See Supplemental Material [ref] for [secondary controls / full error budget / extended data].

### Significance / closing

> Tie the result to the prior belief it changes. A Letter has barely a conclusion —
> do not re-summarize; state the consequence.
[...]

## Figures

> A few high-impact figures only; each costs word-equivalents (budget with the text).
> - Fig. 1 — conveys the central result at a glance (schematic + decisive data).
> - Fig. 2 / 3 — each makes ONE additional point (control / dependence / theory comparison).
> Single-column where possible. Captions define every symbol; keep them tight.

# Cover letter (separate)

> See `prl-cover-letter`. Core job: justify importance + broad interest to the editor.
> 1. Submitted as a Letter to PRL; subject area.
> 2. Central claim (1–2 sentences).
> 3. Why important (belief changed / new effect / unlocked regime).
> 4. Why broad interest (named out-of-subfield audiences + concrete reason).
> 5. Originality; prior arXiv version; suggested/opposed referees.
