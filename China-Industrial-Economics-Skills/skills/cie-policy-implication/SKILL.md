# 政策含义（cie-policy-implication）

## 触发时机

- 文末政策建议是"加强、完善、推进、深化"四件套
- 建议谁都能写，与本文实证发现无关
- 建议停在"意义层"，没说具体怎么做

## 本刊与《经济研究》的关键差异

投稿指南："'启示/政策建议'**来源于结论又高于结论**…体现理论对于实践的指导意义，求新、求深、求实。"本刊办刊理念"**实践立地**"——政策建议必须**可操作**：

> **谁（主体）、在哪个环节（节点）、做什么（动作）、依据本文哪条发现 / 哪个异质性。**

这与《经济研究》"政策含义偏意义层"形成鲜明对比。

## 可操作建议四要素

| 要素 | 反例（不可操作） | 正例（可操作） |
|------|------------------|----------------|
| 主体 | "政府应…" | "试点城市发改部门 / 行业主管部门" |
| 节点 | "加强支持" | "在试点验收的资金拨付环节" |
| 动作 | "完善机制" | "将拨付与企业研发投入增量挂钩" |
| 依据 | （无） | "因本文发现效应集中于融资约束高的企业" |

## 从发现到建议（映射）

- **主效应** → 该政策值得推广 / 调整的总体判断
- **机制** → 建议应作用在机制环节（如经由融资约束 → 配套融资支持）
- **异质性** → **差异化施策**（对哪类主体加力、对哪类退出），这是本刊最看重的政策落点
- 注意边界：本文未检验的不要外推承诺

## 自检清单

- [ ] 每条建议有**主体 + 节点 + 动作**
- [ ] 每条建议**对应本文一条发现 / 一个异质性维度**
- [ ] 有**差异化政策**（来自异质性），不是一刀切
- [ ] 无"加强/完善/推进/深化"空泛四件套
- [ ] 不超出实证证据外推
- [ ] 与摘要、结论的核心观点一致，不重复堆砌

## 反模式

- "建议政府高度重视 X"——谁都能写
- 政策建议与实证发现脱节（"两张皮"）
- 一刀切建议，浪费了异质性结果
- 把"启示"写成对结论的简单复述

## 输出格式

```
【可操作性】高 / 中 / 低
【建议条目】每条：主体 + 节点 + 动作 + 依据发现
【差异化】来自异质性 √ / 一刀切（需改）
【四件套命中】X 处（应为 0）
【外推风险】无 / 有 <点>
【下一步】cie-submission
```
