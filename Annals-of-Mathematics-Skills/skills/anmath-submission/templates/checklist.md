# Annals of Mathematics Pre-Submission Checklist

> Verify all volatile specifics (portal URL, accepted formats, length and cover-letter
> expectations) on the official journal page before relying on this list.

## 1. TeX and Compilation

- [ ] Manuscript is written in TeX using AMS packages (`amsmath`, `amsthm`, `amssymb`)
- [ ] Source compiles cleanly with a standard distribution — no errors or warnings that matter
- [ ] No unresolved cross-references ("??") anywhere in the compiled PDF
- [ ] A single consistent theorem-numbering scheme is used throughout
- [ ] The submitted PDF matches the source to be provided on acceptance

## 2. Front Matter

- [ ] Title and author/affiliation match the portal entries exactly
- [ ] Abstract is self-contained and states the Main Theorem in plain terms
- [ ] MSC primary and secondary classifications match the headline result
- [ ] Keywords (if requested) reflect the area
- [ ] Acknowledgments / funding stated as required

## 3. Main Result and Positioning

- [ ] Main Theorem is stated precisely and appears near the first page
- [ ] The specific advance over prior work is quantified (not just asserted)
- [ ] Closest prior results are cited with correct authors and statements
- [ ] Priority relative to any known preprint/announcement is engaged honestly

## 4. Proof Completeness and Rigor

- [ ] The proof is complete with no logical gaps
- [ ] No unresolved "clearly" / "obviously" / "it is easy to see" hides a step
- [ ] "Standard" / "similar" arguments are named and cited precisely
- [ ] Quantifier order and constant dependence are explicit
- [ ] Every external result invoked is published and precisely cited
- [ ] No essential step depends on an unpublished or unverifiable claim

## 5. Structure and Exposition

- [ ] Every definition/lemma/proposition/theorem is stated before being proved
- [ ] Notation is introduced at first use and used consistently
- [ ] A proof overview precedes the detailed argument
- [ ] Sections are logically ordered with clear purposes
- [ ] Auxiliary/long material is in appendices; nothing essential is hidden there

## 6. Figures and Diagrams (if any)

- [ ] Figures/diagrams are vector format (PDF/EPS) and genuinely necessary
- [ ] Commutative diagrams use `tikz-cd` / `amscd` with consistent, labeled arrows
- [ ] Every figure/diagram is referenced in the text

## 7. References

- [ ] Bibliography uses a consistent style (BibTeX; MathSciNet abbreviations / `\MR`)
- [ ] Every in-text citation appears in the bibliography and vice versa
- [ ] Preprints are labeled; versions of record used where available
- [ ] Citations are accurate (author, year, exact statement relied upon)

## 8. Computer-Assisted and Logistics

- [ ] Any machine computation is documented (software, version, exact claim checked)
- [ ] Code/data for computer-assisted steps are archived and citable
- [ ] arXiv preprint id ready to declare (if posted)
- [ ] Cover letter prepared if the portal requests it
- [ ] File size/format within portal limits; all coauthors approved the final version
