# 修改回复（cfe-rebuttal）

## 触发时机

- 收到《财经研究》R&R 通知
- 已有 1–3 份双盲审稿意见（及编委 / 编辑部意见）
- 准备开始修订正文 + 回复信

## 顺序原则

**先修订正文，再写回复信**。回复信是"导览图"，必须基于已修订的正文版本。

颠倒会导致：
- 回复信承诺了正文未真正做的修订
- 正文修订过程中发现承诺无法兑现
- 多稿来回不一致

## 双盲回复的匿名注意

《财经研究》采用双向匿名评审，回复信也需保持匿名：

- 回复信中**不要暴露作者身份**（不写单位、不写"我们课题组"）
- 引用自己以前的文章时仍用第三人称（"已有研究（XX，2020）"）
- 修订标记用蓝色字体 / 修订模式，但确保最终匿名版无个人信息

## 回复信结构

### 总体结构

```
尊敬的主编、编委、各位审稿专家：

非常感谢编辑部和各位审稿专家在百忙之中对本文的审阅和宝贵意见。
我们已根据各位意见对论文进行了系统修订。下面我们逐条回复各位的意见。
为便于核对，我们在回复中标注了修订对应的正文位置（如"详见正文 P.X 第 Y 段"）。

致主编 / 编委的回复（如有编委意见）
审稿人 1 的回复
审稿人 2 的回复
审稿人 3 的回复（如有）

再次感谢各位的宝贵意见。
作者
20XX 年 XX 月 XX 日
```

### 单条回复结构

```
意见 X：[原文照抄审稿意见]

回复：感谢审稿专家的宝贵意见。我们已[做了什么具体修改]，理由如下：
（1）从理论角度，……
（2）从实证角度，……
（3）从写作角度，……
具体修改详见正文第 X 页第 Y 段（修订标记已用蓝色字体 / 修订模式标出）。
```

## 三种回复模式

### A. 完全接受

- 直接说明接受 + 描述修改 + 指明位置
- 不要找理由绕弯

### B. 部分接受

- 先肯定审稿人的合理性
- 说明部分接受的范围
- 解释未接受部分的原因（**给证据，不要狡辩**）

### C. 不接受（谨慎使用）

- 先充分肯定审稿人的考虑
- 给出严密的反驳论证（至少 3 段）
- 必要时新增稳健性 / 补充分析作为旁证
- **不接受的比例最多 1–2 条**

## 财经研究 R&R 特别注意

- 编委 / 编辑部意见若关于**理论定位 / 与中国现实的关联**，必须重点回应
- 审稿人若要求"补充识别策略稳健性"，应额外加 2–3 个稳健性表格附在文末
- 审稿人若要求"补充机制" → 优先在正文加机制小节，而非仅放附录
- 审稿人若问"为何投本刊 / 学科契合度" → 回应时强调财经方向归属与现实问题

## 必查清单

- [ ] 每条审稿意见都有对应回复（不能遗漏）
- [ ] 每条回复都指明正文修订位置
- [ ] 回复信开头有致谢段
- [ ] 不接受的意见有充分论证 + 旁证
- [ ] 正文修订处有标记（蓝色字体 / 修订模式）
- [ ] 修订版正文已通读，确保无前后矛盾
- [ ] 理论定位 / 现实关联类意见已优先回应
- [ ] 回复信与修订稿均保持匿名（双盲）

## 反模式

- "感谢审稿人的意见，我们已修改"——一行回复
- 回复信比正文还长 —— 多半在凑字数
- 三个审稿人意见冲突，回复信里全部接受 —— 正文会自相矛盾
- 把不接受的回复写得很 defensive
- 忽视编委意见，只回复匿名审稿人
- 回复信中无意间暴露作者身份

## 输出格式

```
【审稿人数】X 人
【意见总数】X 条
【完全接受】X 条
【部分接受】X 条
【不接受】X 条
【回复信字数】X
【正文修订段落数】X
【编委意见回应】完整 / 待补
【匿名合规】通过 / 待修
【下一步】上传至投稿系统 → 等待二轮审稿
```
