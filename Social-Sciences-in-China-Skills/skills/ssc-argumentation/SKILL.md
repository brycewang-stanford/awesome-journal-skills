# 论证与思辨规范（ssc-argumentation）

## 触发时机

- 论证有跳跃，概念界定含糊
- 章节像"专题拼盘"，各节不指向同一命题
- 思辨/规范论证被一堆数据替代，但逻辑链没立住

## 核心要求：每一步服务于核心命题

本刊很多稿是思辨/规范/历史比较型，**逻辑严谨度的权重不低于实证识别**。检验标准：去掉任一节，命题是否塌？若不塌，说明该节是装饰。

## 概念先于论证

- 关键概念**显式界定**：内涵、外延、与近义概念的区分
- 概念在全文含义一致，不中途偷换
- 新造概念要给出"为什么需要它"（旧概念解释不了什么）

## 论证链自检

1. 前提是否明示且可接受？
2. 从前提到结论的每一步推理是否有效？
3. 反例/对立解释是否被认真处理（而非回避）？
4. 结论是否恰好等于已论证的，不过度外推？

## 章节—命题映射表（写作工具）

| 章节 | 它要证明命题的哪一步 | 去掉会塌吗 |
|------|----------------------|------------|
| 第二节 | … | 是/否 |
| 第三节 | … | 是/否 |

> 出现"否"的行，要么删，要么改写到指向命题。

## 自检清单

- [ ] 关键概念有显式界定且全文一致
- [ ] 每节都在推进核心命题（章节—命题映射无"否"）
- [ ] 推理链每步有效，前提明示
- [ ] 认真处理了对立解释/反例
- [ ] 结论不超出已论证范围

## 反模式

- 用案例/数据"举例"冒充"论证"
- 概念中途偷换（如"治理"前后指不同对象）
- 把"很多学者认为"当作论据
- 结论比论证走得远（过度一般化）

## 输出格式

```
【核心命题】<…>
【概念界定】清晰 / 含糊 <概念>
【章节-命题映射】<列出"否"的章节>
【对立解释】已处理 / 回避
【过度外推】有 / 无
【下一步】ssc-evidence 或 ssc-style
```
